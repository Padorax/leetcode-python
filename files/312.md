# Burst Balloons
### Solution DP
打气球（Burst Balloons）是 LeetCode 上最经典的**区间 DP（Interval DP）**题目之一。它的难点在于：如果你按照“戳破”的顺序去思考，每戳破一个气球，剩下的气球就会变成邻居，这会让状态转移变得极其混乱。
核心逆向思维：我们要找的不是“第一个戳破谁”，而是“最后剩下的那个是谁”。
1. 为什么正向思维行不通？
如果你先戳破第 $i$ 个气球，那么 $i-1$ 和 $i+1$ 就变成了邻居。下一次戳气球的得分取决于这次的操作。这种依赖关系让子问题无法独立。
逆向思维：假设在区间 $[i, j]$ 中，最后一个被戳破的气球是 $k$。这意味着在 $k$ 被戳破之前，区间 $[i, k-1]$ 和 $[k+1, j]$ 的气球已经全部被戳破了。此时，$k$ 的左邻居是 $i-1$，右邻居是 $j+1$。
2. 状态定义与转移
为了方便处理边界，我们在原数组前后各加一个 $1$：nums = [1] + nums + [1]。定义 $dp[i][j]$ 为：戳破开区间 $(i, j)$ 内所有气球能获得的最大硬币数。（注意：$i$ 和 $j$ 是边界，不被戳破）。转移方程：在 $(i, j)$ 之间枚举最后戳破的气球 $k$（$i < k < j$）：$$dp[i][j] = \max_{k} \{ \text{左边全部戳破的得分} + \text{右边全部戳破的得分} + \text{最后戳破 k 的得分} \}$$$$dp[i][j] = \max_{k=i+1}^{j-1} \{ dp[i][k] + dp[k][j] + nums[i] \cdot nums[k] \cdot nums[j] \}$$
---
在我们的 $dp[i][j]$ 定义中，$i$ 和 $j$ 是“防弹边界”，它们根本不在被戳破的计划内。
1. 为什么不能戳破 $i$ 或 $j$？
为了让逻辑自洽，我们把问题从“戳破所有气球”变成了：“戳破 $i$ 和 $j$ 之间的所有气球，但保留 $i$ 和 $j$ 作为依靠。”如果我们允许最后戳破的是 $i$（边界），那 $i$ 的左边是谁？邻居关系又会变得模糊。所以，我们通过在原数组两端补 [1]，把原数组的所有气球都变成了“被包围在中间”的气球。
原数组：[3, 1, 5, 8]
补全后：[1, (3, 1, 5, 8), 1]
我们的目标是计算 $dp[0][5]$。这意味着我们戳破索引 $1, 2, 3, 4$ 的气球，而索引 $0$ 和 $5$ 的这两个 1 是永远不会被戳破的。

2. 如果你非要让 $k$ 等于边界会发生什么？
假设你让 $k = i$ 或者是 $k = j$：
* 逻辑冲突：如果你在计算 $dp[i][j]$ 时最后戳破 $i$，那么 $i$ 就不再是这个区间的“固定支点”。
* * 状态重叠：如果最后戳破的是 $i$，那它其实应该是更外层区间（比如 $dp[i-1][j]$）的一个内部节点 $k$。

区间 DP 的精髓就在于： $i$ 和 $j$ 像两堵墙，保护着里面的气球。无论里面怎么炸，墙都不动。只有当这层任务完成了，这两堵墙才可能变成更大区间里的“牺牲品” $k$。

3. 遍历顺序的本质区别 (你之前的思考)你之前问过区间 DP 和线性 DP 的区别，这里就能体现出来：线性 DP (如股票题)：像在画一条线。你从第 1 天画到第 $N$ 天，每一天的状态只取决于它左边的邻居。区间 DP (如打气球)：像在堆金字塔。你先算出所有长度为 1 的小区间（一个气球）；再算出所有长度为 2 的区间（两个气球）；最终堆到长度为 $N$ 的大区间。4. 总结：最后一步的魅力在打气球中，$k$ 必须在 $(i, j)$ 内部。如果你想让第一个气球（索引 1）最后被戳破，你会计算：$dp[0][1] + dp[1][5] + vals[0] \cdot vals[1] \cdot vals[5]$这里的 $dp[0][1]$ 内部没有气球，结果是 $0$。这里的 $dp[1][5]$ 代表戳破中间所有气球后，最后剩下索引 1。这种“先定死两头，再枚举中间”的策略，是解决所有“元素消失后邻居会变”问题的万能钥匙。
---
在打气球问题中，我们不能像线性 DP 那样简单地从左到右遍历，原因就在于：大区间的答案依赖于小区间的答案。
#### 枚举方式1 按长度增长
区间 DP 的特点是：先枚举区间长度，再枚举左端点。
为什么 dp[i][k] 和 dp[k][j] 一定已经算好了？观察我们的转移方程：$$dp[i][j] = \max_{k} \{ dp[i][k] + dp[k][j] + \text{score} \}$$$dp[i][k]$ 的区间长度是 $k - i$。$dp[k][j]$ 的区间长度是 $j - k$。我们要算的 $dp[i][j]$ 的区间长度是 $j - i$。因为 $k$ 是在 $i$ 和 $j$ 之间（$i < k < j$），所以 $dp[i][k]$ 和 $dp[k][j]$ 的区间长度一定都比 $dp[i][j]$ 短。结论： 只要我们按照**“区间长度从小到大”**的顺序来计算，那么在算大区间时，它内部所有的子区间（更短的区间）都已经计算完毕并存储在 DP 表中了。
```python
class Solution:
    def maxCoins(self, nums: List[int]) -> int:
        # 补上边界
        vals = [1] + nums + [1]
        n = len(vals)
        # dp[i][j] 表示戳破 (i, j) 之间气球的最大值
        dp = [[0] * n for _ in range(n)]
        
        # length 是区间内气球的数量，从 1 到 n-2
        for length in range(1, n - 1):
            # 枚举左边界 i
            for i in range(n - length - 1):
                j = i + length + 1 # 右边界 j
                # 枚举最后戳破的气球 k
                for k in range(i + 1, j):
                    score = vals[i] * vals[k] * vals[j] # vals[k] not nums[k]
                    dp[i][j] = max(dp[i][j], dp[i][k] + dp[k][j] + score)
                    
        return dp[0][n-1]
```

#### 枚举方式2
写法 B：从下往上，从左往右（更紧凑）如果你观察 DP 矩阵，你会发现 dp[i][j] 依赖于它左边的值（$dp[k][j]$，行不变，列变小）和它下面的值（$dp[i][k]$，列不变，行变大）。所以我们要保证：$i$ 从大到小遍历，而 $j$ 从小到大遍历。
```python
class Solution:
    def maxCoins(self, nums: List[int]) -> int:
        vals = [1] + nums + [1]
        n = len(vals)
        dp = [[0] * n for _ in range(n)]

        for i in range(n - 2, -1, -1):          # i 从下往上
            for j in range(i + 2, n):           # j 从左往右
                for k in range(i + 1, j):
                    dp[i][j] = max(dp[i][j], dp[i][k] + dp[k][j] + vals[i]*vals[k]*vals[j])
        return dp[0][n-1]
```

当你发现一个问题有以下特征时，就要考虑区间 DP：
* 消除类问题（如打气球、石子合并、祖玛游戏）。
* 操作顺序影响后续邻居关系。
* 可以分解为小区间的最优解叠加。
