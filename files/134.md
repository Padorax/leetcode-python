```python
class Solution:
    def canCompleteCircuit(self, gas, cost):
        """
        :type gas: List[int]
        :type cost: List[int]
        :rtype: int
        """
        start = len(gas) - 1
        end = 0
        sum = gas[start] - cost[start]
        while (start > end):
            if sum>=0 :
                sum += gas[end] - cost[end]
                end += 1
            else:
                start -= 1
                sum += gas[start] - cost[start]
        #return start if sum>=0 else -1
        if sum >= 0:
            return start
        else:
            return -1
```

or
我们用i，j分别标记起点和终点，用s表示当前剩余汽油，而cnt表示当前行驶过的加油站数量。初始时，我们将起点设在最后一个位置，即 i=n-1开始行驶，移动j, 若发现当前剩余汽油小于0, 说明当前i作为起点不符合要求，我们将点i循环左移，并且更新剩余汽油，直至剩余汽油是非负数。

当行驶过的加油站数量达到n时，结束行驶过程。若此时剩余汽油s仍然小于0, 说明不存在这样的起点，返回-1。否则，返回起点i。

time complexity: O(n), space: O(1)

```python
class Solution:
    def canCompleteCircuit(self, gas: List[int], cost: List[int]) -> int:
        n = len(gas)
        i = j = n - 1
        cnt = s = 0
        while cnt < n:
            s += gas[j] - cost[j]
            cnt += 1
            j = (j + 1) % n
            while s < 0 and cnt < n:
                i -= 1
                s += gas[i] - cost[i]
                cnt += 1
        return -1 if s < 0 else i
```

### Solution 3 Greedy (easy)
加油站（Gas Station）是一个非常经典的**贪心算法**问题。虽然你可以用 $O(n^2)$ 的暴力法尝试从每个起点出发，但通过一点数学直觉，我们可以把它优化到 $O(n)$。

* 两个关键结论
1. 总油量必须 > 总消耗
如果你把全城所有的油加起来，还不够跑完一圈的总消耗，那么无论你从哪儿出发，最终都会抛锚。
2. 一旦“断油”，起点必在后方
假设你从 `A` 出发，一路顺风顺水到了 `B`，但在去 `C` 的路上油箱空了。
* 这说明：从 `A` 到 `B` 之间的**任何一个加油站**作为起点，都不可能到达 `C`。
* **原因**：你从 `A` 出发到达中间站时，油箱里多少还剩点油（或者是 0），这总比你直接从中间站“白手起家”（油箱为 0）要强。如果连带着剩余油量的 `A` 都死在了去 `C` 的路上，那从中间站出发只会死得更早。

* 算法实现逻辑
基于这两个结论，我们的算法只需要**遍历一次**：
1. 初始化 `total_tank` (总剩余) 和 `curr_tank` (当前油箱)。
2. 遍历每一个站：
* 更新 `total_tank` 和 `curr_tank`（加上 $gas[i] - cost[i]$）。
* 如果 `curr_tank < 0`：说明当前的起点不行。
* **重置**：把下一个站 `i + 1` 设为新起点。
* **清空** `curr_tank`。
3. 最后检查 `total_tank`：如果它小于 0，返回 -1；否则返回记录的起点。

```python
class Solution:
    def canCompleteCircuit(self, gas: List[int], cost: List[int]) -> int:
        total_tank, curr_tank = 0, 0
        starting_station = 0
        
        for i in range(len(gas)):
            diff = gas[i] - cost[i]
            total_tank += diff
            curr_tank += diff
            
            # 如果在 i 站油耗尽了
            if curr_tank < 0:
                # 贪心：起点至少得是 i+1
                starting_station = i + 1
                curr_tank = 0
                
        return starting_station if total_tank >= 0 else -1

```
* 为什么这个算法是正确的？

你可能会问：“我怎么知道从 `starting_station` 出发一定能跑完剩下的半圈并回到它自己呢？”

这就是**结论 A** 的魅力：
如果你能保证 `total_tank >= 0`，并且你找到一个起点能跑完**后面**所有的站（即 `curr_tank` 一直 $\ge 0$ 直到数组末尾），那么根据能量守恒，它一定能补上数组前面那些“亏欠”的油量，顺利绕回起点。

* **对比动态规划**：之前的“打气球”或“编辑距离”是在构建复杂的选择树，而这道题通过**剪枝**（一旦断油，跳过中间所有站）直接变成了一次线性扫描。
* **对比双指针**：这道题其实也可以用双指针（快慢指针）来模拟环形路径，但贪心思路显然更优雅。
