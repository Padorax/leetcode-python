# Minimum Money Required Before Transactinos
要求：在进行任何一笔交易时，balance > cost, and 如果有下一笔交易：new balance = balance - cost(i)+cashback(i) > cost(i+1)
观察：最后剩余资金一定>=0 (因为在进行any tx including last one之前，balance >= cost(i), then we have rewards which is >= 0)

1. 如果给定消费顺序，如何确定min初始资金？
e.g., [[2, 1], [5, 0], [4, 2]]

### solution 1 - 逆向
我们知道在进行最后一笔tx [4, 2]时资金应该 >= 4, then在前一笔时，资金=max(5, 5-0+4)=9, then初始资金=max(2, 2-1+9)=10
```python
def minMoney(transactions):
    res = transactions[-1][0]
    for i in range(len(transactions)-2, -1, 0):
        cost, cashback = transactions[i]
        res = max(cost, cost-cashback+res)
    return res
```
### Solution 2 - offset
假设初始balance为0， 求出每一步的余额，我们只需要记录最大负余额， 再加一个offset把它抬升到>=0即可
e.g., 初始0 -> -2 (支出2) -> -1 (cashback 1) -> -6 (支出5) -> -6 -> -10 (支出4) -> -8 (don't care here as it's the last tx)， 最“负”余额为10， 所以我们需要初始balance为10
```python
def minMoney(transactions):
    minBalance = 0
    balance = 0
    for cost, reward in transactions:
        minBalance = min(balance - cost, minBalance)
        balance += reward-cost
        # or 
        # balance -= cost
        # minBalance = min(minBalance, balance)
        # balance += reward
    return -minBalance
```

2. 
2.1 如何确定一个消费顺序，使得按这个特定顺序需要的初始资金最少？
对于收益为负的交易（cost > cashback，即亏本任务）：
    应该尽可能晚地做。因为这些任务会减少我们的总余额，如果放在前面做，会导致后续任务对余额要求更高。
    排序方式：按 cashback（即交易后的余额）从大到小处理。

    假设你有两个亏本任务 A(c1, r1) 和 B(c2, r2)：如果你先做 A 后做 B，要求初始资金至少为 max(c1, c1 - r1 + c2)。如果你先做 B 后做 A，要求初始资金至少为 max(c2, c2 - r2 + c1)。通过代数对比可以发现，当 r2 > r1（即 B 的回馈更大）时，先做 B 往往能让初始资金要求更低。

对于收益为正的交易（cost <= cashback，即赚钱任务）：
    应该尽可能早地做。因为这些任务会增加我们的总余额，做完后钱变多了，更容易满足后续任务的要求。
    排序方式：按 cost（即支付金额）从小到大处理。

```python
def minInitialMoneyOptimalOrder(transactions):
    # 1. 分类
    profit_tasks = [t for t in transactions if t[0] <= t[1]]
    loss_tasks = [t for t in transactions if t[0] > t[1]]
    
    # 2. 排序
    # 赚钱的：先做门槛(cost)低的
    profit_tasks.sort(key=lambda x: x[0])
    # 亏钱的：先做回血(cashback)多的
    loss_tasks.sort(key=lambda x: x[1], reverse=True)
    
    # 3. 合并顺序
    optimal_order = profit_tasks + loss_tasks
    
    # 4. 模拟特定顺序下的最低余额
    current_balance = 0
    min_balance = 0
    for cost, cashback in optimal_order:
        current_balance -= cost
        min_balance = min(min_balance, current_balance)
        current_balance += cashback
        
    return abs(min_balance)
```

2.2
！！反之，如果需要无论任何顺序下初始资金都满足条件 - 则应该先做收益为负的，使minBalance尽可能降低， 再做收益为正的。对于收益为负的，先做reward小的；对于收益为正的，先做cost大的

```python
class Solution:
    def minimumMoney(self, transactions: List[List[int]]) -> int:
        total_loss = 0
        max_extra = 0
        
        for cost, cashback in transactions:
            if cost > cashback:
                # 累加所有亏损交易的净亏损
                total_loss += (cost - cashback)
                # 如果这是最后一笔亏损交易，我们需要 TotalLoss - (cost - cashback) + cost
                # 简化后就是 TotalLoss + cashback
                max_extra = max(max_extra, cashback)
            else:
                # 如果这是在所有亏损后进行的一笔盈利交易
                # 我们需要在当前余额不足 cost 时补足即可 TotalLoss + cost
                max_extra = max(max_extra, cost)
                
        return total_loss + max_extra
```
