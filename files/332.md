# Reconstruct Itinerary
### Solution 1 DFS - TLE
We must build an itinerary that:
- starts from "JFK"
- uses every ticket exactly once
- is lexicographically smallest among all valid itineraries.
This DFS solution tries destinations in sorted order.
At each airport src, we choose one outgoing ticket, remove it (so it can't be reused), and continue DFS.
If we reach a dead end before using all tickets, we backtrack: undo the choice and try the next destination.

Sorting tickets ensures the first complete valid path we find is the smallest lexicographically.

1. Sort tickets lexicographically.
2. Build an adjacency list adj[src] = list of destinations in sorted order.
3. Start res = ["JFK"].
4. Run DFS from "JFK":
* If len(res) == len(tickets) + 1, all tickets are used → return true.
* For each possible destination v from src (in order):
  * Remove that edge (src -> v) from adj[src] (use the ticket).
  * Append v to res.
  * If DFS from v succeeds, return true.
  * Otherwise backtrack: Remove v from res
Insert the destination back into adj[src] at the same position.
5. Return res.
```python
class Solution:
    def findItinerary(self, tickets: List[List[str]]) -> List[str]:
        adj = {src: [] for src, dst in tickets}
        tickets.sort()
        for src, dst in tickets:
            adj[src].append(dst)

        res = ["JFK"]
        def dfs(src):
            if len(res) == len(tickets) + 1:
                return True
            if src not in adj:
                return False

            temp = list(adj[src])
            for i, v in enumerate(temp):
                adj[src].pop(i)
                res.append(v)
                if dfs(v): return True
                adj[src].insert(i, v)
                res.pop()
            return False

        dfs("JFK")
        return res
```

```python
from collections import defaultdict
class Solution:
    def findItinerary(self, tickets: List[List[str]]) -> List[str]:
        # 1. 必须先排序，保证 DFS 找到的第一个解就是字典序最小的
        tickets.sort()
        g = defaultdict(list)
        for src, dst in tickets:
            g[src].append(dst)
        
        self.res = [] # 【修改 1】使用 self 确保在 DFS 内部可以访问/修改
        def dfs(path):
            if len(path) == len(tickets) + 1:
                self.res = path # 【修改 2】赋值给外部变量
                return True
            curr = path[-1] # 【修改 3】定义当前出发点
            # 如果当前机场没有出港航班，直接返回 False
            if curr not in g: return False
            # 【修改 4】直接遍历邻接表，不要在循环里做深拷贝（太慢）
            for i in range(len(g[curr])):
                # caveat!! in normal dfs we don't revisit a node
                # here we can revisit a node, but cannot re-use an edge
                dest = g[curr].pop(0)          # 【修改 5】取出当前的航线
                if dfs(path + [dest]):         # 递归
                    return True
                g[curr].insert(i, dest)        # 【回溯】放回原位，保证顺序不变
            return False

        dfs(["JFK"])
        return self.res
```

```python
from collections import defaultdict
class Solution:
    def findItinerary(self, tickets: List[List[str]]) -> List[str]:
        tickets.sort()
        g = defaultdict(list)
        for src, dst in tickets:
            g[src].append(dst)
        
        self.res = []
        def visit(path, visited, airport):
            if len(path) == len(tickets) + 1:
                self.res = path
                return
            for i, nxt in enumerate(g[airport]):
                if (airport, i) not in visited:
                    # record i-th edge of airport (this can deal with duplicate edge of current airport)
                    visited.add((airport, i))
                    visit(path + [nxt], visited, nxt)
                    if self.res: # stop at first result as it's lexicographically smallest
                        return 
                    visited.remove((airport, i))
        visit(["JFK"], set(), "JFK")
        return self.res
```
### HierHolzer - 欧拉通路
```python
class Solution:
    def findItinerary(self, tickets: List[List[str]]) -> List[str]:
        # 1. 构建邻接表，并进行逆序排序
        # 为什么要逆序？因为下面用 pop() 弹出末尾元素是 O(1)，比 pop(0) 快得多
        g = defaultdict(list)
        for src, dst in sorted(tickets, reverse=True):
            g[src].append(dst)
        
        route = []

        def visit(airport):
            # 2. 只要当前机场还有没飞过的票，就一直飞下去
            while g[airport]:
                next_dest = g[airport].pop()
                visit(next_dest)
            
            # 3. 关键点：当无路可走时，才将当前机场加入结果
            # 这意味着“最后无路可走”的机场会最先被加入列表
            route.append(airport)

        visit("JFK")
        
        # 4. 因为是倒序加入的，最后翻转一下就是完整的行程
        return route[::-1]
```
```python
def findItinerary(self, tickets):
    targets = collections.defaultdict(list)
    for a, b in sorted(tickets)[::-1]:
        targets[a] += b,
    route = []
    def visit(airport):
        while targets[airport]:
            visit(targets[airport].pop())
        route.append(airport)
    visit('JFK')
    return route[::-1]
```

想象你在走一个复杂的迷宫，迷宫里有很多环路。如果你只是普通的 DFS，你可能会进了一个环路出不来，或者走到了终点却发现还有别的环没逛。
Hierholzer 算法的聪明之处在于：它先让你一条路走到黑（直到死胡同），然后回溯。在回溯的过程中，如果发现某个点还有没走过的“环”，它会顺便把那个环也“画”完，再接着往回走。

# 对比总结
1. 核心区别：点的访问 vs 边的消耗
在普通的拓扑排序或 DFS 中，我们通常关注的是点（Node）。但在欧拉路径（一笔画）中，我们关注的是边（Edge）。

普通 DFS：如果走错了路（先去了 C），你就“死”在了 C，必须通过 Backtracking（回溯） 撤销这个选择，重新回到 A 去试 B。

Hierholzer (后序)：它不撤销。它说：“既然 C 是死胡同，那 C 肯定是全行程的终点，我把它先记在小本本的最后一行，然后我回到 A 继续逛没逛完的 B 环。”

2. 拆解你的例子：A → B, B → A, A → C
假设我们不考虑字典序，且 DFS 运气不好，第一步就选了 A -> C：

visit(A): 选择了 A -> C。

visit(C): 发现 C 没有出港机票了（死胡同）。

关键动作：将 C 加入结果 route = [C]。

函数返回到 visit(A)。

回到 visit(A): A 发现自己还有一张票 A -> B 没用。

调用 visit(B)。

visit(B): 发现有一张票 B -> A。

调用 visit(A) (第三次进入)。

visit(A) (第三次): A 没票了。

关键动作：将 A 加入结果 route = [C, A]。

函数返回。

回到 visit(B): B 没票了。

关键动作：将 B 加入结果 route = [C, A, B]。

函数返回。

回到 visit(A) (第一次): A 的所有票都用光了。

关键动作：将 A 加入结果 route = [C, A, B, A]。

最后翻转结果： [A, B, A, C]。
你看！即便第一步选错了（选了去 C），算法依然通过“后序入栈”把 C 巧妙地挤到了最后，而把 A-B-A 这个环补在了前面。

3. 为什么拓扑排序（Kahn's Algorithm）在这里不适用？
拓扑排序是处理 DAG（有向无环图） 的，它的目的是确定先后顺序（比如选课）。

拓扑排序：是为了避开环。

欧拉路径：是为了吃掉所有的环。

如果图中存在 A -> B -> A 这样的环，拓扑排序的入度统计就会卡死（入度永远不为 0），而 Hierholzer 正好是利用递归栈把这些环一个个“套”进主路径里的。

总结：
你说的“后序”是神来之笔。Hierholzer 本质上就是：

先处理“子问题”（剩下的路），最后才把“当前点”定死在路径里。

这种“走到头再回头”的思路，在处理复杂的路径搜索时，比反复 pop/insert 的回溯要高效得多。
