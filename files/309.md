# Best Time to Buy and Sell Stock with Cooldown
You are given an array prices where prices[i] is the price of a given stock on the ith day.

Find the maximum profit you can achieve. You may complete as many transactions as you like (i.e., buy one and sell one share of the stock multiple times) with the following restrictions:

After you sell your stock, you cannot buy stock on the next day (i.e., cooldown one day).
Note: You may not engage in multiple transactions simultaneously (i.e., you must sell the stock before you buy again).

Example 1:
Input: prices = [1,2,3,0,2]
Output: 3
Explanation: transactions = [buy, sell, cooldown, buy, sell]

Example 2:
Input: prices = [1]
Output: 0

### Solution DP (state transition)
为了清晰地处理冷冻期，我们可以把状态拆解为三种，并观察它们每天是如何转换的：
1. 持有状态 (Hold)：手里有一张票。
    来源：昨天就持有的，或者昨天是“冷冻期满”今天买入的。
2. 卖出状态 (Sold)：今天刚卖掉。
   来源：昨天一定处于“持有状态”，今天把它卖了。
3. 冷冻期/休息状态 (Rest)：今天没票，且不是今天卖的。（ready to buy)
   来源：昨天也是休息状态，或者昨天刚卖完今天被迫休息。

我们定义 dp[i]为第i天结束时的最大利润：
* Hold[i]: max(Hold[i-1], Rest[i-1] - prices[i])
  解释：要么继续持仓，要么今天买入（买入的前提是昨天得是休息状态)
* Sold[i]: Hold[i-1] + prices[i]
  解释：今天卖了，利润就是昨天的持仓成本加上今天的价格。
* Rest[i]: max(Rest[i-1], Sold[i-1])   
  解释：今天休息，要么昨天就休息，要么昨天刚卖完。
hold - sold - rest -(hold)

举个例子假设价格是 [1, 2, 3, 0, 2]：
Day 1: 买入 (Hold = -1)
Day 2: 卖出 (Sold = 1)
Day 3: 冷冻期 (Rest = 1)。由于冷冻期存在，你不能在 Day 3 买入。
Day 4: 价格跌到 0。此时你可以买入了！因为 Day 3 是休息状态。(Hold = 1 - 0 = 1)Day 5: 价格回升到 2。卖出。(Sold = 1 + 2 = 3)最终利润是 3。
```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        if not prices:
            return 0
        
        # 初始化
        # hold: 第一天买入，利润为负数
        # sold: 第一天不可能卖出，设为 0
        # rest: 第一天不买，利润为 0
        hold = -prices[0]
        sold = 0
        rest = 0
        
        for i in range(1, len(prices)):
            prev_hold = hold
            prev_sold = sold
            prev_rest = rest
            
            # 状态更新
            hold = max(prev_hold, prev_rest - prices[i])
            sold = prev_hold + prices[i]
            rest = max(prev_rest, prev_sold)
            
        # 最后的最大利润一定不在 hold 状态（手里有票没卖肯定不是最优）
        return max(sold, rest)
```

# 拓展
若cool down days = 2，则需要增加一个状态：
1. Hold: 手中持有股票。
2. Sold: 今天刚刚卖出股票（触发冷冻期第 1 天）。
3. Cooldown 1: 卖出后的第 1 天（仍然不能买，进入冷冻期第 2 天）。
4. Rest/Ready: 冷冻期结束，可以买入，或者继续观望。
设 dp[i] 为第 i 天各状态的最大利润：
Hold[i]: max(Hold[i-1], Ready[i-1] - prices[i])
    解释：要么继续持仓，要么在“准备好了”的状态下买入。
Sold[i]: Hold[i-1] + prices[i]
    解释：今天卖出。
Cool1[i]: Sold[i-1]
    解释：昨天刚卖，今天强制进入第一级冷冻。
Ready[i]: max(Ready[i-1], Cool1[i-1])
    解释：昨天已经冷冻期满了，或者昨天是冷冻的最后一天，今天终于解禁
```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        if not prices: return 0
        
        hold = -prices[0]
        sold = 0
        cool1 = 0
        ready = 0
        
        for i in range(1, len(prices)):
            # 记录旧状态
            p_hold, p_sold, p_cool1, p_ready = hold, sold, cool1, ready
            
            # 更新当前状态
            hold = max(p_hold, p_ready - prices[i])
            sold = p_hold + prices[i]
            cool1 = p_sold
            ready = max(p_ready, p_cool1)
            
        return max(sold, cool1, ready)
```
如果冷冻期变为 M 天，你需要 M 个类似的 Cool 状态来充当“延迟计数器”。从 Sold 开始，利润像接力棒一样经过 $Cool_1, Cool_2, ..., Cool_M。只有走完了整个接力赛，这笔钱才能到达 Ready 状态，重新被允许用来购买股票。这种设计本质上是在模拟时间窗口的平移。

## M天冷冻期通式
当冷冻期变成变量 M 时，我们不再手动写 cool1, cool2，而是用一个队列（Queue）或者延迟数组来处理利润的“流动”。逻辑：卖出后的利润不能立刻变现，必须在 cooldown_queue 里排队，走完 M 步后才能进入 ready 状态。
```python
def maxProfit(prices: List[int], m: int) -> int:
    if not prices: return 0
    
    hold = -prices[0]
    # cooldowns 记录卖出后每一天状态的利润
    # 长度为 m，代表卖出后必须走过 m 天
    cooldowns = [0] * m 
    ready = 0
    sold = 0

    for i in range(1, len(prices)):
        prev_hold = hold
        prev_ready = ready
        
        # 1. 今天的 hold：要么继续持有，要么从 ready 状态买入
        hold = max(hold, ready - prices[i])
        
        # 2. 今天的 sold：今天卖出的利润
        today_sold = prev_hold + prices[i]
        
        # 3. 今天的 ready：上一天的 ready 或 已经走完 m 天冷冻期的利润
        # 也就是 cooldowns 队列的最后一个元素释放出来了
        ready = max(ready, cooldowns[-1] if m > 0 else sold)
        
        # 4. 更新冷冻队列：today_sold 进，末尾出
        if m > 0:
            # 利润在冷冻期内“移动”
            cooldowns = [today_sold] + cooldowns[:-1]
        else:
            sold = today_sold
            
    return max(ready, sold, *cooldowns)
```

### 含手续费
手续费其实比冷冻期更简单。它不限制你的“时间”，只限制你的“收益”。
策略选择：你只需在卖出或买入时，扣除一次 fee 即可。通常建议在卖出时扣除，逻辑更自然。
状态方程：
* Hold[i] = max(Hold[i-1], NotHold[i-1] - prices[i])
* NotHold[i] = max(NotHold[i-1], Hold[i-1] + prices[i] - fee)
