# Meeting Rooms II
Given an array of meeting time intervals intervals where intervals[i] = [starti, endi], return the minimum number of conference rooms required.

 Example 1:
Input: intervals = [[0,30],[5,10],[15,20]]
Output: 2

Example 2:
Input: intervals = [[7,10],[2,4]]
Output: 1

### Solution 1 minHeap
```python
class Solution:
    def minMeetingRooms(self, intervals: List[Interval]) -> int:
        res = [] # each element represents a room
        intervals.sort(key=lambda x: x.start)
        for itvl in intervals:
            if not res:
                res.append([itvl]) # 1st room
                continue
            
            useExisting = False
            for room in res:
                last = room[-1]
                # can use existing room
                if itvl.start >= last.end:
                    room.append(itvl)
                    useExisting = True
                    break
            # start a new room
            if not useExisting:
                res.append([itvl])
        return len(res)
```
With optimization (only record latest meeting end time of a room)

```python
class Solution:

        res = []
        intervals.sort(key=lambda x: x[0])
        for itvl in intervals:
            if not res:
                res.append(itvl[1])
                continue
            
            useExisting = False
            for i, end in enumerate(res):
                if itvl[0] >= end:
                    res[i] = itvl[1]
                    useExisting = True
                    break
            if not useExisting:
                res.append(itvl[1])
        return len(res)
```

instead of iterating over existing rooms, use a minHeap to record earliest ending meeting room
```python
class Solution:
    def minMeetingRooms(self, intervals: List[Interval]) -> int:
        intervals.sort(key=lambda x: x.start)
        min_heap = []

        for interval in intervals:
            if min_heap and min_heap[0] <= interval.start:
                heapq.heappop(min_heap)
            heapq.heappush(min_heap, interval.end)

        return len(min_heap)
```
time: O(nlogn), space: O(n)

### Solution chronological ordering上下车解法/2 pointers
想象会议室是一个公交车，会议开始就是“上车”，会议结束就是“下车”。我们关心的只是同一时刻车上最多有多少人。(or同一时刻最多有多少同时进行的回忆)
```python
class Solution:
    def minMeetingRooms(self, intervals: List[Interval]) -> int:
        starts = sorted([i.start for i in intervals])
        ends = sorted([i.end for i in intervals])
        
        res = count = 0
        s_ptr = e_ptr = 0
        #每当 s_ptr 准备“处理”下一个会议时，它都会先看一眼 e_ptr：
        #1. 清算阶段：如果 ends[e_ptr] 小于当前 starts[s_ptr]，说明在你要开始这个新会议之前，已经有会议结束了。算法会通过 e_ptr 的移动，把所有“已经结束但还没注销”的房间全部扣除（count -= 1）。
        #2. 入场阶段：直到所有比当前时间早的结束事件都处理完了，s_ptr 才会执行 count += 1
        #入场阶段：直到所有比当前时间早的结束事件都处理完了，s_ptr 才会执行 count += 1
        while s_ptr < len(intervals):
            if starts[s_ptr] < ends[e_ptr]:
                # 有人要开会，且还没人结束，必须开新房间
                count += 1
                # s_ptr的最后一次处理过后一定得到当前有几个正在同时进行的会议
                s_ptr += 1
            else:
                # 有人开完会了，腾出一个房间
                count -= 1
                e_ptr += 1 # note s_ptr stays the same to eliminate all finished meetings
            res = max(res, count)
        return res
```
nit: since process only triggered by start event, we will not decrease after last "start" happened (does not matter here since we are tracing max not min)
- count will not go down to 0 in the end, but in scanning line it will (as "end" is also processing trigger )
### Sweep/scanning line
这段代码不再关注具体的某个会议，而是关注时间轴上的每一个关键点（开始或结束）对房间需求量的改变。
* mp[i.start] += 1：代表在 start 这个时刻，房间的需求量增加了 1 个。
* mp[i.end] -= 1：代表在 end 这个时刻，房间的需求量减少了 1 个（释放了一个房间）
```python
class Solution:
    def minMeetingRooms(self, intervals: List[Interval]) -> int:
        mp = defaultdict(int)
        for i in intervals:
            mp[i.start] += 1
            mp[i.end] -= 1
        prev = 0
        res = 0
        for i in sorted(mp.keys()):
            prev += mp[i]
            res = max(res, prev)
        return res
```

```python
class Solution:
    def minMeetingRooms(self, intervals: List[List[int]]) -> int:
        
        events=list()
        for start,end in intervals:
            events.append([start,+1])
            events.append([end,-1])

        events.sort(key=lambda x: (x[0],x[1])) #nit: end comes before start for same time (so (1, 2), (2, 1) will process end (-1) then start (+1), do not increase room at overlapping point)
        ans=active=0
        for _,delta in events:
            active+=delta
            ans=max(ans,active)

        return ans
```
